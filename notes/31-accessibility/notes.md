# CSS Accessibility

## 1. Concept

CSS must support accessible user experiences.

---

## 2. Key Principles

- readable text
- good contrast
- clear focus states

---

## 3. Example

```css
button:focus {
  outline: 2px solid blue;
}
```

---

## 4. Important Areas

- color contrast
- font size
- spacing
- animations

---

## 5. Best Practices

- avoid removing focus outlines
- ensure readable font sizes
- limit motion

CSS can improve or harm accessibility depending on usage :contentReference[oaicite:3]{index=3}

---

## 6. Mental Model

Accessibility = usability for everyone

---

## 7. Common Mistakes

- low contrast text
- tiny fonts
- excessive animations

## 9. Summary

Accessible CSS ensures your UI works for all users.
